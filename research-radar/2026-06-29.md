## What we chose to build, and why

**Two artifacts this week**, both *tools* rather than paper reimplementations.
The bar — *real, current pain* + *fully testable in one sitting* + *clean-room*
— eliminated the headline papers (which need an LLM API in the loop) in favor
of two small, sharply-scoped primitives that every team writes a buggy version
of:

1. **`jsonlcheck`** — strict streaming JSONL validator with line:column
   diagnostics. Direct response to the 2026-current pain (sources [2] [3])
   that JSONL is the dominant fine-tuning / eval-log format yet existing
   tooling silently accepts NaN / Infinity / duplicate keys / BOMs that
   corrupt training runs days later. 53 tests; zero dependencies.

2. **`seedline`** — one opinionated `seed_all(n)` + a restore-on-exit
   `seeded(n)` context manager, covering Python, NumPy, and PyTorch (when
   present). Every ML repo carries a homegrown version of this, and most
   have at least one bug (forgotten backend, post-start `PYTHONHASHSEED`,
   accepting `True` as a seed). 24 tests; zero hard dependencies.

Both ship under MIT, with honest scope-and-limits sections, and were built
clean-room (not derived from any existing tool's source).

We deliberately deferred everything LLM-API-dependent (test-suite humanization,
agent governance harnesses) — those need a hosted model and a budget plan
before they can be honestly tested and would not have cleared the quality bar
this run.

## Signal we noted (not built this week)

1. **Govern the Repository, Not the Agent — repo-level integration friction
   from autonomous coding agents** (arXiv 2606.28235). Quantifies friction
   across 930K+ PRs. The shift from agent-evaluation to ecosystem-evaluation
   is the more durable framing — most current eval benchmarks measure agents
   in isolation. Worth a benchmark-harness extraction later; needs PR-corpus
   access to be honest. [arXiv](https://arxiv.org/abs/2606.28235)

2. **Humanizing automatically-generated unit tests via LLM refactoring**
   (arXiv 2606.28229). Hybrid SBST + LLM with a controlled refactoring layer.
   Real problem; not reimplementable this run because the test-refactoring
   step needs a hosted LLM. Candidate for a *deterministic* sub-component
   (the refactoring legality checks) on a future week. [arXiv](https://arxiv.org/abs/2606.28229)

3. **Surprises in Proper Positive-Only Learning** (arXiv 2606.28309).
   Characterizes proper learnability of concept classes from positive-only
   samples (finite VC + uniform exterior separability). Theoretical, no
   clean software artifact yet. [arXiv](https://arxiv.org/abs/2606.28309)

4. **Non-Linear Strategic Classification Made Practical** (arXiv 2606.28204).
   Strategic best-response via Lagrangian duality. Could yield a small
   reference implementation if paired with a public dataset — bookmark.
   [arXiv](https://arxiv.org/abs/2606.28204)

5. **Show HN pattern: dashboards-as-code for agents** ("DAC"). The recurring
   2026 friction — agentic systems that have to drive UI dashboards via a
   browser — keeps producing new "dashboard as YAML" tools. The pain is
   real; existing entrants are crowded. [HN 47949066](https://news.ycombinator.com/item?id=47949066)

6. **Ask HN: What developer tool do you wish existed in 2026?** Confirmed
   pain: IDEs fighting over the same git, agent task-context switching,
   project mixing in IDE windows. Tractable angles for later weeks: a tiny
   per-worktree git-lockfile arbiter; a `git worktree`-aware status board.
   [HN 46345827](https://news.ycombinator.com/item?id=46345827)

7. **LLM Structured Outputs: Schema Validation for Real Pipelines (2026)**
   — documents the gap between "constrained-decoding gives you valid JSON"
   and "your *downstream* validator catches the semantic and cross-field
   errors that constrained decoding can't." Direct evidence that the
   JSONL-side validation layer (where `jsonlcheck` sits) is still patchy
   in 2026. [collinwilkins.com](https://collinwilkins.com/articles/structured-output)

8. **arXiv `daily_papers` / Papers-with-Code state**: the strategy note from
   2026-06-27 in the ops repo flagged Papers-with-Code's effective shutdown
   and arXiv API 429-rate-limiting; this scan confirmed both still hold —
   adapters need backoff and graceful fallback (the radar must ship even
   when a feed dies).

- Developer-tool pain points via targeted query.

  this run?* — by far the most exclusionary filter.
